How to prepare sequences for dbBact querying

Sequence trimming

dbBact uses 16S sequences from the V1, V3 or V4 variable regions. Sequences should start immediately after the end of the relevant region primer:
* V1: AGRGTTTGATYMTGGCTCAG
* V3: CCTACGGGNBGCWSCAG
* V4: GTGCCAGCMGCCGCGGTAA

For example, given the sequence:
gctaactac gtgccagcagccgcggtaa TACGTAGGGGGCAAGCGTTATCCGGATTTACTGGGTGTAAAGGGAGCGTAGACGGCGAAGCAAGTCTGAAGTGAAAACCCAGGGCTCAACCCTGGGACTGCTTTGGAAACTGTTTTGCTAGAGTGTCGGAGAGGTAAGTGGAATTCCTAG
We should only use the sequence following the V4 primer (i.e. the green sequence following the blue primer sequence) for dbBact queries

NOTE:When querying the dbBact website with a single sequence, it automatically tries to trim primers if present in the sequence

Query sequence length

Sequences should be of length at least 100 nucleotides (but can be any length >100 as dbBact sequences shorter/longer than the query sequence will still be identified as match)
For example, the dbBact sequence:
TACGTAGGGGGCAAGCGTTATCCGGATTTACTGGGTGTAAAGGGAGCGTAGACGGCGAAGCAAGTCTGAAGTGAAAACCCAGGGCTCAACCCTGGGACTGCTTTGGAAACTGTTTTGCTAGAGTGTCGGAGAGGTA
Will be matched by the (shorter) query sequence:
TACGTAGGGGGCAAGCGTTATCCGGATTTACTGGGTGTAAAGGGAGCGTAGACGGCGAAGCAAGTCTGAAGTGAAAACCCAGGGCTCAACCCTGGGACTGCTTTGGAAACTGTTTTGCTAGAGTG
And by the (longer) query sequence:
TACGTAGGGGGCAAGCGTTATCCGGATTTACTGGGTGTAAAGGGAGCGTAGACGGCGAAGCAAGTCTGAAGTGAAAACCCAGGGCTCAACCCTGGGACTGCTTTGGAAACTGTTTTGCTAGAGTGTCGGAGAGGTAAGTGGAATTCCTAG

Inter-region sequence matching

dbBact uses the SILVA 16S dataset for inter-region sequence matching. If a query sequence is identical to a subsequence of a SILVA sequence, annotations regarding other regions of the SILVA sequence will also be identified.
For example, querying the V4 sequence:
TACGTAGGGGGCAAGCGTTATCCGGATTTACTGGGTGTAAAGGGAGCGTAGACGGCGAAGCAAGTCTGAAGTGAAAACCCAGGGCTCAACCCTGGGACTGCTTTGGAAACTGTTTTGCTAGAGTGTCGGAGAGGTAAGTGGAATTCCTAG
Matches the SILVA [clostridium] clostridioforme sequence (ID kr364750)
This SILVA sequence contains the V3 sequence:
TGGGGAATATTGCACAATGGGCGAAAGCCTGATGCAGCGACGCCGCGTGAGTGAAGAAGTATTTCGGTATGTAAAGCTCTATCAGCAGGGAAGAAAATGACGGTACCTGACTAAGAAGCCCCGGCTAACTACGTGCCAGCAGCCGCGGTA
And therefore dbBact annotations matching this V3 sequence will also be obtained when querying using the V4 sequence.

Additional requirements

Sequences should contain only A/C/G/T
